# 全量工具函数缺失列表

> 基于当前仓库实际已注册的工具包能力梳理。  
> 对照参考主要是 Lightroom Develop 核心链路 + Photoshop 高频修图能力。  
> 这份清单同时包含：
> 1. 缺失的**基础工具函数（primitive）**
> 2. 缺失的**复合功能（workflow / macro）**
> 3. 开发优先级建议

## 1. 当前已具备的工具能力

当前 registry 已注册的工具包有：

- `adjust_exposure`
- `adjust_highlights_shadows`
- `adjust_contrast`
- `adjust_whites_blacks`
- `adjust_curves`
- `adjust_clarity`
- `adjust_texture`
- `adjust_dehaze`
- `adjust_color_mixer`
- `adjust_white_balance`
- `adjust_vibrance_saturation`
- `crop_and_straighten`
- `denoise`
- `sharpen`

这些能力已经覆盖了 Lightroom 中最核心的参数化修图链路：

- 基础明暗：曝光 / 高光阴影 / 白场黑场 / 对比度 / 曲线
- 色彩基础：白平衡 / 自然饱和度 / HSL 风格颜色混合
- 细节链：清晰度 / 纹理 / 去灰雾 / 降噪 / 锐化
- 构图链：裁剪 / 拉直
- 局部能力：动态 mask 驱动的局部参数调整

结论：

- 当前系统已经具备“参数化修图引擎”的基本骨架
- 但距离“Lightroom + 一部分 Photoshop 高频修图能力”还差几块关键能力

## 2. 缺失的基础工具函数（Primitive）

下面是建议补齐的**底层能力**。这些能力更适合先做成独立工具函数，再考虑组合成复合功能。

### 2.1 修复 / 去瑕疵类

这类能力当前是最明显的缺口。

| 功能 | 建议工具名 | 价值 | 优先级 | 备注 |
|---|---|---:|---:|---|
| 智能去除 / 修补 | `remove_heal` | 很高 | P0 | 去痘、去灰尘、去路人、去小杂物、水印残留 |
| 克隆修复 | `clone_stamp` | 高 | P1 | 更适合规则纹理区域；可作为 heal 的补充 |
| 内容识别去除 | `content_aware_remove` | 高 | P1 | 更接近 Photoshop Remove / Generative Remove 的体验 |
| 修复画笔局部替换 | `spot_heal` | 中高 | P1 | 用于小面积瑕疵、皮肤杂点、传感器灰尘 |
| 反射去除 / 眩光削弱 | `reflection_reduce` | 中高 | P1 | 玻璃反射、镜片反光、局部 glare |

### 2.2 人像基础修饰类

这类功能是“自然人像修图”最容易被用户直接感知的能力。

| 功能 | 建议工具名 | 价值 | 优先级 | 备注 |
|---|---|---:|---:|---|
| 皮肤柔化 | `skin_smooth` | 很高 | P0 | 比单纯 clarity/texture 更稳定 |
| 肤质纹理减弱 | `skin_texture_reduce` | 高 | P1 | 更细粒度控制，比 `skin_smooth` 更底层 |
| 痘痘 / 瑕疵去除 | `blemish_remove` | 很高 | P0 | 高需求高价值 |
| 眼下提亮 | `under_eye_brighten` | 高 | P1 | 常见人像精修动作 |
| 牙齿美白 | `teeth_whiten` | 中高 | P1 | 高频但局部范围较小 |
| 眼睛提亮 / 瞳孔增强 | `eye_brighten` | 中高 | P1 | 人像收尾非常常见 |
| 唇色增强 | `lip_enhance` | 中 | P2 | 更偏人像商业图 |
| 发丝提亮 / 发丝质感增强 | `hair_enhance` | 中高 | P1 | 比锐化更专用，更适合人像 |

### 2.3 镜头 / 光学 / 几何类

这类能力是 Lightroom 中很标准、但当前系统基本空缺的部分。

| 功能 | 建议工具名 | 价值 | 优先级 | 备注 |
|---|---|---:|---:|---|
| 镜头畸变校正 | `lens_correction` | 高 | P1 | 桶形/枕形失真 |
| 色差去除 | `remove_chromatic_aberration` | 高 | P1 | 逆光边缘紫边 |
| 去边色 / Defringe | `defringe` | 高 | P1 | 发丝边缘、白衣边缘很常见 |
| 透视校正 | `perspective_correction` | 高 | P1 | 建筑、室内、商品图常见 |
| 自动 Upright | `auto_upright` | 中高 | P2 | 可与 perspective correction 组合 |
| 暗角控制 | `vignette` | 中高 | P2 | 氛围感与边缘收束常用 |
| 颗粒 | `grain` | 中 | P2 | 胶片风格的重要组成 |
| 摩尔纹抑制 | `moire_reduce` | 中 | P2 | 衣料、屏幕、建筑网格等 |

### 2.4 颜色 / 调色进阶类

你们现在有传统 HSL 风格 Color Mixer，但还缺更精细的颜色工具。

| 功能 | 建议工具名 | 价值 | 优先级 | 备注 |
|---|---|---:|---:|---|
| Point Color 精准颜色选择 | `point_color` | 很高 | P1 | 比传统 HSL 更适合“白裙去黄”“肤色微调” |
| 色彩分级 / Color Grading | `color_grading` | 高 | P1 | 阴影/中间调/高光调色轮 |
| LUT 应用 | `apply_lut` | 中高 | P2 | 适合快速风格化 |
| B&W 转换 | `convert_black_white` | 中 | P2 | Lightroom 常规能力 |
| Camera Calibration | `camera_calibration` | 中 | P3 | 专业调色方向 |

### 2.5 景深 / 模糊 / 气氛类

| 功能 | 建议工具名 | 价值 | 优先级 | 备注 |
|---|---|---:|---:|---|
| 背景虚化 | `background_blur` | 高 | P1 | 对人像、商品、小红书风格都很值 |
| 镜头虚化 / 景深模拟 | `lens_blur` | 高 | P1 | 比普通 Gaussian blur 更接近摄影感 |
| 局部径向模糊 / 运动模糊 | `motion_blur_local` | 中 | P3 | 风格化场景 |
| Bloom / Glow | `glow_highlight` | 中高 | P2 | 婚礼、夜景、梦幻风格常见 |

### 2.6 遮罩 / 选区进阶类

虽然系统现在已经有动态 prompt 分割，但还缺一些非常实用的“非语义遮罩”。

| 功能 | 建议工具名 | 价值 | 优先级 | 备注 |
|---|---|---:|---:|---|
| 线性渐变蒙版 | `mask_linear_gradient` | 很高 | P1 | Lightroom 高频工具 |
| 径向渐变蒙版 | `mask_radial_gradient` | 很高 | P1 | 局部打光、聚焦中心很常见 |
| 画笔蒙版 | `mask_brush` | 高 | P1 | 手工局部细修 |
| 亮度范围蒙版 | `mask_luminance_range` | 高 | P1 | 控高光、控暗部非常好用 |
| 颜色范围蒙版 | `mask_color_range` | 高 | P1 | 白裙、天空、草地、肤色更稳 |
| 深度蒙版 | `mask_depth` | 中高 | P2 | 适合景深、背景分离 |

### 2.7 生成式 / 替换类

如果目标不是只做 Lightroom 能力，而是“Lightroom + Photoshop 高频能力”，这些也属于缺失项。

| 功能 | 建议工具名 | 价值 | 优先级 | 备注 |
|---|---|---:|---:|---|
| 局部 inpaint 替换 | `inpaint_replace` | 很高 | P1 | 去路人、去物体、改背景局部 |
| 扩图 / Outpaint | `outpaint_expand` | 高 | P2 | 社媒封面、横竖版扩展 |
| 背景替换 | `replace_background` | 高 | P2 | 高频，但风险高 |
| 文本替换 / 区域重写 | `rewrite_region` | 中高 | P2 | 广告图、海报图更有价值 |

### 2.8 形变 / 人体结构类（偏 Photoshop）

| 功能 | 建议工具名 | 价值 | 优先级 | 备注 |
|---|---|---:|---:|---|
| 液化 / 局部形变 | `liquify_warp` | 高 | P2 | 人像、服装、电商都有需求 |
| 面部微调 | `face_reshape` | 中高 | P2 | 鼻梁、脸型、嘴角等 |
| 身材比例微调 | `body_reshape` | 中 | P3 | 风险高，要谨慎 |

## 3. 缺失的复合功能（Macro / Composite）

这些功能不建议一开始直接做成低层 primitive，而更适合建立在现有参数工具 + 新增 primitive 之上。

### 3.1 人像类复合功能

| 功能 | 推荐名称 | 依赖基础能力 | 优先级 | 备注 |
|---|---|---:|---:|---|
| 自然美白 | `portrait_natural_whitening` | 皮肤 mask + 曝光 + 白平衡 + point_color + skin_smooth | P0 | “美白”最好做成自然净透，而不是刷白 |
| 肤色净透 | `portrait_skin_clean_tone` | 曝光 + 色温 + point_color + texture/clarty 反向微调 | P0 | 比“美白”更自然 |
| 人像精修 | `portrait_retouch` | under_eye / skin / hair / teeth / eye / blemish | P1 | 高频商业需求 |
| 逆光人像修复 | `portrait_backlight_repair` | 主体提亮 + 高光回收 + 背景压制 + 肤色平衡 | P0 | 非常高频 |
| 婚纱保护增强 | `wedding_dress_protect` | 白裙 mask + whites/highlights + point_color + texture protect | P0 | 当前案例高频出现 |
| 发丝质感增强 | `portrait_hair_detail_boost` | hair mask + texture/sharpen + fringe control | P1 | 常用收尾能力 |

### 3.2 氛围 / 风格类复合功能

| 功能 | 推荐名称 | 依赖基础能力 | 优先级 | 备注 |
|---|---|---:|---:|---|
| 夏日通透感 | `summer_airy_look` | white_balance + vibrance + dehaze + point_color + background control | P0 | 当前目标场景高频 |
| 日系清透 | `japanese_clean_portrait` | 低对比 + 高亮保护 + 肤色净透 + 微冷白平衡 | P1 | 小红书/人像常见 |
| 胶片暖调 | `film_warm_tone` | curve + color_grading + grain + vignette | P1 | 风格化高频 |
| 夜景氛围感 | `night_mood_boost` | shadow control + highlights + color grading + glow | P2 | 城市夜景很值 |
| 商品高光质感增强 | `product_specular_enhance` | 局部 clarity/texture/sharpen + highlight control | P1 | 瓶子、水珠、玻璃类很适合 |

### 3.3 修复类复合功能

| 功能 | 推荐名称 | 依赖基础能力 | 优先级 | 备注 |
|---|---|---:|---:|---|
| 去路人 | `remove_passersby` | segmentation + remove/heal/inpaint | P1 | 高频刚需 |
| 去痘 / 去瑕疵 | `cleanup_skin_blemishes` | blemish_remove + skin_smooth | P0 | 高频刚需 |
| 去杂物 / 去脏点 | `cleanup_distracting_objects` | remove/heal | P1 | 真实图片里很常见 |

## 6. 针对当前系统最现实的下一步建议

如果只从“短期最能补齐缺口”的角度看，推荐开发顺序是：

1. **修复类 primitive**
   - `remove_heal`
   - `blemish_remove`
   - `skin_smooth`

2. **颜色精细化 primitive**
   - `point_color`

3. **白裙 / 肤色 / 逆光的复合能力**
   - `portrait_natural_whitening`
   - `portrait_backlight_repair`
   - `wedding_dress_protect`
   - `summer_airy_look`

4. **镜头与边缘控制**
   - `remove_chromatic_aberration`
   - `defringe`
   - `lens_correction`

5. **遮罩进阶**
   - `mask_linear_gradient`
   - `mask_radial_gradient`
   - `mask_luminance_range`
   - `mask_color_range`

## 7. 总结

当前系统的主要问题不是“完全没有修图能力”，而是：

- **能调色调光，但缺少修复类能力**
- **能做基础人像调整，但缺少人像精修 primitive**
- **能做传统 HSL，但缺少更精准的颜色范围控制**
- **能做动态语义 mask，但缺少 Lightroom 式高频遮罩工具**
- **缺少高频复合能力（自然美白、逆光修复、白裙保护、夏日通透感）**

如果目标是快速补齐“用户会明显感知到的缺口”，优先级最高的是：

- 修复类
- 人像 skin primitive
- point color
- 复合功能（自然美白 / 逆光修复 / 白裙保护 / 夏日通透感）
