# 当前工具能力与待完成工具清单

> 基于当前仓库真实已注册的工具包能力整理。  
> 对照参考主要是 **Adobe Lightroom Classic / Camera Raw 的 Develop 能力** 与 **Adobe Photoshop 高频修图能力**。  
> 本文档的目标是回答三件事：
> 1. 现在系统已经能做什么
> 2. 还缺什么工具函数
> 3. 哪些应该先做成底层工具，哪些更适合做成复合功能

---

## 1. 当前系统的工具能力总览

当前 registry 中实际注册的工具包有：

- `adjust_exposure`
- `adjust_highlights_shadows`
- `adjust_contrast`
- `adjust_whites_blacks`
- `adjust_curves`
- `adjust_clarity`
- `adjust_texture`
- `adjust_dehaze`
- `adjust_color_mixer`
- `adjust_white_balance`
- `adjust_vibrance_saturation`
- `crop_and_straighten`
- `denoise`
- `sharpen`

当前系统整体定位已经不是“零能力修图原型”，而是：

- 具备 **Lightroom / Camera Raw 核心参数化修图骨架**
- 具备 **动态遮罩驱动的局部调整能力**
- 具备 **一定的人像/主体/局部精修基础**
- 但还不具备 Photoshop 高频修复能力与更高级的颜色/人像精修能力

## 2. 当前已具备的工具能力（按 Lightroom / PS 习惯归类）

## 2.1 构图与几何类

### `crop_and_straighten`

**能力定位**
- 对应 Lightroom 的 `Crop & Straighten`
- 用于：
  - 裁剪构图
  - 校正轻微倾斜
  - 画面边缘安全裁切

**当前支持**
- 全图模式
- 强度映射为裁剪比例
- 拉直角度控制
- 裁剪安全边距
- 最小保留比例

**当前不足**
- 没有透视校正
- 没有建筑 Upright
- 没有镜头畸变校正

---

## 2.2 基础明暗链（Lightroom Basic / Tone）

### `adjust_exposure`

**能力定位**
- 对应 Lightroom / ACR 的 `Exposure`
- 用于整体提亮/压暗，或者局部区域亮度修正

**当前支持**
- 全图 / 遮罩局部
- 抽象强度映射为曝光档位（stops）
- 支持羽化半径

**典型适用**
- 主体偏暗
- 逆光人像补亮
- 局部压暗过亮区域

**当前不足**
- 没有自动曝光智能参考
- 没有基于高亮/阴影保护的更高级曝光策略

### `adjust_highlights_shadows`

**能力定位**
- 对应 Lightroom 的 `Highlights / Shadows`
- 用于拉回高光、提暗部细节

**当前支持**
- 全图 / 遮罩局部
- 阴影量、高光量、局部亮度估计半径
- 中间调对比补偿
- tonal width 控制
- 细节回灌

**典型适用**
- 逆光脸部提亮
- 白裙高光细节保护
- 暗部层次恢复

**当前不足**
- 还没有真正的亮度范围 mask 联动
- 高光保护 / 白裙保护仍依赖 planner 组合调用

### `adjust_whites_blacks`

**能力定位**
- 对应 Lightroom 的 `Whites / Blacks`
- 用于设定白点和黑点

**当前支持**
- 全图 / 遮罩局部
- 白场黑场独立控制
- highlight / shadow rolloff

**典型适用**
- 提升画面通透度
- 建立明暗落点
- 背景压暗、主体亮部干净化

**当前不足**
- 没有更高级的直方图参考式自动建议
- 没有高亮物体/白裙专用保护模式

### `adjust_contrast`

**能力定位**
- 对应 Lightroom 的 `Contrast`
- 用于围绕中灰点增加/降低对比

**当前支持**
- 全图 / 遮罩局部
- 中灰锚点
- 高光保护
- 阴影保护

**当前不足**
- 仍偏整体对比操作
- 对人像脸部和皮肤的细致控制不如曲线 + 局部 mask

### `adjust_curves`

**能力定位**
- 对应 Lightroom / Photoshop 的 `Tone Curve`
- 用于精细塑造整体或局部明暗关系

**当前支持**
- 全图 / 遮罩局部
- 阴影提拉
- 中间调 gamma
- 高光压缩
- 整体对比偏置

**典型适用**
- 基础校正之后的小幅塑形
- 提升层次而不显脏
- 婚纱/肤色/背景的精细 tone 调整

**当前不足**
- 还不是控制点曲线
- 没有 RGB 通道曲线
- 不能做更高级色偏修正

---

## 2.3 白平衡与基础色彩链

### `adjust_white_balance`

**能力定位**
- 对应 Lightroom / ACR 的 `Temp / Tint`
- 用于整体或局部纠正冷暖和偏色

**当前支持**
- 全图 / 遮罩局部
- 温度映射
- 色调偏移（绿-洋红）
- 高饱和颜色保护

**典型适用**
- 偏黄、偏蓝、偏绿
- 逆光人像肤色校正
- 白裙 / 白墙 / 白物体中和偏色

**当前不足**
- 没有更高级的灰点/白点参考
- 没有 color balance / selective color 那种阴影、中间调、高光分段校色

### `adjust_vibrance_saturation`

**能力定位**
- 对应 Lightroom 的 `Vibrance / Saturation`
- 用于自然增色或整体降饱和

**当前支持**
- 全图 / 遮罩局部
- vibrance 与 saturation 分离
- 高光 / 肤色 / 暗部保护
- 色度去噪
- 中性色保护

**典型适用**
- 让画面更有存在感
- 夏日感、清透感
- 避免肤色直接过饱和

**当前不足**
- 缺少更高级的 color grading
- 缺少胶片风格里的颗粒和分区色调

### `adjust_color_mixer`

**能力定位**
- 对应 Lightroom 的 `Color Mixer / HSL`
- 按颜色组做 hue / saturation / luminance 调整

**当前支持**
- red / orange / yellow / green / aqua / blue / purple / magenta
- 各颜色的 hue / saturation / luminance
- 全图 / 遮罩局部

**典型适用**
- 肤色偏红 / 偏黄修正
- 草地、天空、白裙、暖色物体单独微调
- 风格化颜色整理

**当前不足**
- 这是传统 HSL，不是 Point Color
- 颜色范围还是偏宽
- 对“只想改肤色但不动背景暖色”的精度不够

## 2.4 Presence / 细节链

### `adjust_clarity`

**能力定位**
- 对应 Lightroom 的 `Clarity`
- 用于增强中间调局部对比

**当前支持**
- 全图 / 遮罩局部
- 半径倍率
- 高光/阴影保护

**典型适用**
- 提升主体层次
- 让背景更有空气感
- 商品局部轮廓更立体

**当前不足**
- 容易让皮肤变硬
- 没有专门的人像皮肤安全模式

### `adjust_texture`

**能力定位**
- 对应 Lightroom 的 `Texture`
- 用于中尺度纹理增强/柔化

**当前支持**
- 全图 / 遮罩局部
- 纹理尺度倍率
- 噪点保护

**典型适用**
- 头发、布料、水花、玻璃、石头、木纹
- 也可反向减弱纹理

**当前不足**
- 没有人像皮肤纹理专用模式
- 没有更高层的 `skin_smooth / texture reduce` 封装

### `adjust_dehaze`

**能力定位**
- 对应 Lightroom 的 `Dehaze`
- 用于压掉灰雾、恢复通透感

**当前支持**
- 全图 / 遮罩局部
- 亮度保护
- 颜色保护

**典型适用**
- 远景空气层
- 背景灰雾
- 需要增强通透感的场景

**当前不足**
- 没有背景专用去灰雾宏
- 容易和对比/清晰度叠加后发脏，仍依赖 planner 判断

### `denoise`

**能力定位**
- 对应 Lightroom / ACR 的 `Noise Reduction`
- 用于亮度与色度噪点控制

**当前支持**
- 全图 / 遮罩局部
- luma / chroma 分离
- detail protection
- 窗口大小控制

**当前不足**
- 没有 AI denoise
- 没有按 ISO/噪点估计自动推荐

### `sharpen`

**能力定位**
- 对应 Lightroom / ACR 的 `Sharpening`
- 用于边缘和细节锐化

**当前支持**
- 全图 / 遮罩局部
- amount / radius / threshold
- 高光保护

**当前不足**
- 没有边缘蒙版（masking）参数
- 没有人像五官/头发专用 sharpen 宏

---

## 2.5 当前系统的能力边界总结

### 当前已经比较完善的部分

- 参数化基础修图主链
- 局部 mask 驱动的参数调整
- 常见 Lightroom 基础滑块
- 基础色彩与细节链

### 当前最明显的缺口

- Photoshop 高频修复类能力
- 人像肤质/瑕疵类能力
- 更精细的颜色选择能力（Point Color / Selective Color / Color Balance）
- 镜头校正和边缘缺陷处理
- 遮罩进阶工具（渐变、颜色范围、亮度范围）
- 生成式修复和替换能力

## 7. 总结

当前系统已经具备：

- Lightroom 核心参数化修图主链
- 动态遮罩驱动的局部参数调整
- 基础色彩、细节和构图能力

当前最缺的，不是再多加一两个滑块，而是：

- **修复类能力**
- **人像 skin / blemish / eye / teeth 类能力**
- **更精细的颜色选择能力**
- **镜头与边缘缺陷能力**
- **Lightroom 式进阶遮罩能力**
- **高频复合功能（自然美白、逆光修复、婚纱保护、夏日通透感）**

如果只从“最能让用户明显感知系统变强”的角度看，优先级最高的是：

1. `remove_heal`
2. `blemish_remove`
3. `skin_smooth`
4. `point_color`
5. `portrait_natural_whitening`
6. `portrait_backlight_repair`
7. `wedding_dress_protect`
8. `summer_airy_look`
